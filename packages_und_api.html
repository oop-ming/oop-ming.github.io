Packages und API

Einführung - Ein Ordnersystem für Klassen

Ein Package in Java ist wie ein digitaler Ordner für deinen Code. Stell dir vor, du hast viele verschiedene Notizen für deine Uni-Kurse. Um den Überblick zu behalten, sortierst du sie in verschiedene Ordner - einen für Mathematik, einen für Geschichte und so weiter. Packages in Java funktionieren ähnlich.

Hier sind die wichtigsten Punkte:

  1. Organisation: Packages helfen dir, deine Java-Klassen zu gruppieren und zu organisieren. Alle Klassen, die zusammengehören, kommen in ein Package.
  2. Namensgebung: Packages haben Namen wie java.util oder com.mycompany.project. Diese Namen helfen, Konflikte zu vermeiden, wenn verschiedene Programmierer ähnliche Klassennamen verwenden.
  3. Zugriffskontrolle: Packages ermöglichen es dir, den Zugriff auf deine Klassen zu kontrollieren. Du kannst entscheiden, welche Teile deines Codes von außerhalb des Packages sichtbar sein sollen.
  4. Standardbibliothek: Java selbst nutzt Packages, um seine Standardfunktionen zu organisieren. Zum Beispiel findest du viele nützliche Klassen im Package java.util
  5. Verwendung: Um eine Klasse aus einem Package zu nutzen, musst du entweder den vollen Namen angeben (wie java.util.ArrayList) oder eine Import-Anweisung am Anfang deiner Datei schreiben.

Packages sind also im Grunde ein Weg, deinen Code sauber und übersichtlich zu halten, ähnlich wie du deine Uni-Unterlagen organisierst

In diesem Kapitel lernst du:

  • Was die Java API ist
  • Wie man Pakete und Klassen importiert
  • Wie man selber Pakete und Klassen definieren kann.

Java API

APIs (Application Programming Interfaces) sind sogenannte Programmierschnittstellen. Sie bestehen aus mehreren Definitionen und Protokollen zur Softwareentwicklung. Die Java API ist eine umfassende Sammlung von Bibliotheken und Packages. Sie bildet die Standardbibliothek von Java und enthält unzählige Module, welche wiederum Packages enthalten. Die Java API ist automatisch in deiner Java Entwicklungsumgebung integriert.

Wichtige Punkte zur Java API:

  1. Sie umfasst eine Vielzahl von Modulen, wie z.B. java.base, java.desktop, und java.xml
  2. Jedes Modul enthält mehrere Packages, die thematisch zusammengehörige Klassen und Interfaces gruppieren
  3. Die Java SE 17 API enthält rund 4.400 Typen, verteilt auf verschiedene Packages
  4. Einige wichtige Packages der Java API sind:
    • java.util: Hilfsklassen für Datenstrukturen und Algorithmen
    • java.io: Ein- und Ausgabeoperationen
    • java.net: Netzwerkprogrammierung
    • java.lang: Grundlegende Sprachunterstützung

Die Java API wird als Teil der Java Development Kit (JDK) Distribution bereitgestellt und ist somit eine integrierte Sammlung von Bibliotheken, die standardmäßig mit Java verfügbar sind. Das bedeutet du kannst entweder einzelne Klassen oder ganze Pakete (die mehrere Klassen enthalten) importieren. Eine Liste mit allen Paketen und Klassen innerhalb der Java API findest du auf Oracle.

Importieren von Klassen und Packages

Den Syntax zum Einbinden bzw. Importieren eines Pakets kennen wir bereits aus Processing. Es ist derselbe Befehl, der zum einbinden einer Bibliothek verwendet wird:

import package.name.Class;   // Klasse importieren
import package.name.*;          //  Paket importieren

Ein konkretes Beispiel ist das Importieren des Collection Frameworks aus dem Kapitel Collections und Lambda Ausdrücke:

import java.util.ArrayList;   // Importiert die Klasse ArrayList
import java.util.*;                // Importiert alle Collections

Erstellen eigener Pakete

Am Anfang des Kapitels haben wir gesagt, dass Packages Klassen gruppieren und man sich das vorstellen kann wie Ordner in einem Dateisystem. Tatsächlich kann man sich das nicht nur so vorstellen – es ist auch so. Java nutzt ein Dateisystem, um Packages zu speichern. Das könnte zum Beispiel so aussehen:

  • Haupt-Verzeichnis
    • meinPaket
      • MeineKlasse

Um ein Package zu erstellen, musst du also eigentlich nur in deinem src Ordner anlegen, der einen Namen hat, der den Syntaxregeln von Java entspricht, wie z.B. meinPaket. In diesem Ordner kannst du dann Klassen erstellen. Diese Klassen müssen in der ersten Zeile den Befehl package name; (wobei name der Name deines Pakets ist) enthalten (z.B. package meinPaket;). Dann sind diese Klassen in diesem Paket organisiert.

Aber ... so einfach geht das nicht. 

Wenn du in IntelliJ mit der rechten Maustaste auf den src Ordner klickst und dann New auswählst, dann findest du hier nicht die Option Folder (Ordern), sondern nur Java Class und Package (und ein paar weitere Optionen). Wenn du die Option Package wählst, übernimmt IntelliJ die ganze Arbeit für dich.

Um ein Package zu erstellen hast du in IntelliJ zwei Möglichkeiten. In beiden Fällen klickst du mit der rechten Maustaste auf dein src Ordner wählst dann ...

  • New > Package (nur das Paket erstellen)
    In diesem Fall gibst du den Namen deines Pakets unter Beachtung der Bezeichner-Regeln in Java ein. Sobald du einen Namen eingegeben und die Eingabe bestätigt hast, erstellt IntelliJ einen neuen Ordner im src Ordner. Wenn du diesen Ordner nun rechts klickst und in dem Ordner eine neue Java Klasse anlegst, dann wird in der ersten Zeile der Befehl package name; (wobei name der Name deines Pakets ist) stehen.
  • New > Java Class (Paket und Klasse erstellen in einem Schritt)
    Wer es etwas schneller haben möchte, kann die oberen beiden Schritte (erst das Paket, dann die Klasse erstellen) in einem Schritt machen, indem man die Option Java Class wählt und hier im Eingabefeld den Paketnamen und den Klassennamen getrennt durch einen Punkt angibt (z. B. meinPaket.MeineKlasse). Dann erstellt IntelliJ erst das Paket (und den Ordner) meinPaket und direkt danach die Klasse MeineKlasse. Auch hier findest du in der ersten Zeile innerhalb der Klasse den package Befehl.

Zusammenfassung

  • Packages gruppieren Klassen in einer sinnvollen Struktur und verhindern Konflikte bei der Namensgebung
  • Es gibt vordefinierte und benutzerdefinierte Pakete
  • Die Java API ist eine mächtige Bibliothek mit vielen Klassen, die du kostenlos nutzen kannst
  • Pakete und Klassen werden mithilfe von import importiert
  • Eigene Pakete können über das User-Interface von IntelliJ und mithilfe des package Schlüsselworts erstellt werden.
