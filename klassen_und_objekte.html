Klassen und Objekte

Einführung – Von Bauplänen zu realen Objekten

Stell dir vor, du möchtest ein individuelles Auto bauen. Du hast einen exakten Bauplan, der alle Komponenten des Autos beschreibt – Motor, Karosserie, Räder und Innenausstattung. Dieser Bauplan enthält alle Informationen, die du benötigst, um ein Auto zu produzieren, jedoch ist er selbst noch kein fahrbares Fahrzeug. In der objektorientierten Programmierung (OOP) entspricht dieser Bauplan einer Klasse.

Einmal erstellt, kannst du anhand dieses Bauplans beliebig viele Autos (Objekte) fertigen – jedes Auto wird zwar nach demselben Konzept gebaut, besitzt jedoch seine eigenen Eigenschaften (z. B. Farbe, Baujahr) und verhält sich in bestimmten Situationen individuell. Genau so verhält es sich in der OOP: Eine Klasse definiert die Struktur und das Verhalten (Methoden), während Objekte konkrete Instanzen dieser Klasse sind. Diese Metapher – vom abstrakten Bauplan zum realen Produkt – soll dir helfen, den grundlegenden Zusammenhang zwischen Klassen und Objekten zu verstehen.

In diesem Kapitel wirst du lernen:

  • Was Klassen und Objekte sind
  • Wie du Klassen erstellst und Objekte instanziierst
  • Welche Rolle Attribute, Methoden und Konstruktoren spielen
  • Wie Kapselung (Datenverbergen) zur Sicherung der integrierten Daten beiträgt
  • Wie Zugriffsmodifizierer, Getter und Setter den Zugriff steuern und kontrollieren
  • Und wie Packages und APIs zur Strukturierung größerer Programme beitragen

1. Definition von Klassen

In Java beschreibt eine Klasse den Bauplan eines Datentyps. Eine Klasse enthält:

Attribute (Felder): Diese speichern den Zustand eines Objekts. Bei einem Auto könnten das z. B. die Marke, das Modell und das Baujahr sein.
Methoden: Diese definieren das Verhalten, das Objekte ausführen können. Zum Beispiel könnte eine Methode zum Starten des Motors oder Bremsen gedacht sein.
Die Definition einer Klasse legt also nicht nur fest, wie ein Objekt aussieht, sondern auch, welche Aktionen mit ihm ausgeführt werden können. Hier ein einfaches Beispiel einer Klasse „Auto“:

public class Auto {
   // Attribute – der Zustand eines Autos
   private String marke;
   private String modell;
   private int baujahr;

   // Konstruktor: Erzeugt ein neues Auto-Objekt
   public Auto(String marke, String modell, int baujahr) {
      this.marke = marke;
      this.modell = modell;
      this.baujahr = baujahr;
   }

   // Methode: Startet das Auto
   public void starten() {
      System.out.println("Das Auto startet.");
   }

   // Getter und Setter für das Attribut 'marke'
   public String getMarke() {
      return marke;
   }

   public void setMarke(String marke) {
      this.marke = marke;
   }

}



In diesem Beispiel fungiert die Klasse Auto als Bauplan. Sie legt fest, welche Eigenschaften (marke, modell, baujahr) ein Auto besitzt und welche Aktionen (z. B. starten) durchgeführt werden können.


2. Erstellen von Objekten

Sobald eine Klasse definiert ist, kannst du Objekte davon instanziieren. Ein Objekt ist eine konkrete Ausprägung der Klasse, vergleichbar mit einem fertigen Auto, das auf Basis des Auto-Bauplans hergestellt wurde.

So könnte in der Main-Methode eines Programms ein Auto-Objekt erzeugt und verwendet werden:

public class Main {
   public static void main(String[] args) {
      // Erzeugen eines Auto-Objekts
      Auto meinAuto = new Auto("VW", "Golf", 2020);
      meinAuto.starten(); // Ruft die Methode 'starten' auf
      System.out.println("Marke: " + meinAuto.getMarke());
   }
}

Durch den Konstruktor der Klasse wird das Objekt „meinAuto“ erzeugt und mit individuellen Eigenschaften ausgestattet.


3. Attribute und Methoden

Attribute und Methoden bilden das Herzstück einer Klasse:

  • Attribute (Felder) definieren den Zustand. Sie können verschiedene Datentypen haben (z. B. int, String, boolean).
  • Methoden definieren das Verhalten eines Objekts. Sie enthalten Code, der Operationen auf den Attributen oder sonstige Aktionen durchführt.

Ein anschauliches Beispiel:
Stell dir einen „Student“ vor. Er hat Attribute wie Name, Matrikelnummer und Studiengang. Gleichzeitig kann er Aktionen ausführen wie zur Vorlesung gehen oder sich für Prüfungen anmelden. In Java könnte dies folgendermaßen modelliert werden:

public class Student {
   private String name;
   private int matrikelnummer;
   private String studiengang;

   // Konstruktor
   public Student(String name, int matrikelnummer, String studiengang) {
      this.name = name;
      this.matrikelnummer = matrikelnummer;
      this.studiengang = studiengang;
   }

   // Eine Methode: Der Student meldet sich an
      public void anmelden() {
      System.out.println(name + " hat sich für den Kurs angemeldet.");
   }

   // Getter und Setter
   public String getName() {
      return name;
   }

   public void setName(String name) {
      this.name = name;
   }

}


Diese Trennung von Zustand (Attribute) und Verhalten (Methoden) ist ein zentrales Merkmal der OOP. Durch die klare Strukturierung kann der Code übersichtlicher und modularer gestaltet werden.

4. Konstruktoren

Ein Konstruktor ist eine spezielle Methode, die beim Erzeugen eines Objekts aufgerufen wird, um es zu initialisieren. Im obigen Beispiel wird durch den Konstruktor der Klasse Auto sicher gestellt, dass beim Erzeugen eines Auto-Objekts die Eigenschaften marke, modell und baujahr gesetzt werden.

Wichtige Aspekte von Konstruktoren:

  • Namen: Der Konstruktor hat immer denselben Namen wie die Klasse.
  • Keine Rückgabe: Konstruktoren haben keinen Rückgabetyp, auch nicht void.
  • Überladung: Es ist möglich, mehrere Konstruktoren mit unterschiedlichen Parameterlisten zu definieren, um Objekte flexibel initialisieren zu können.

Beispiel – Überladene Konstruktoren:

public class Auto {
   private String marke;
   private String modell;
   private int baujahr;

   // Konstruktor mit allen Parametern
   public Auto(String marke, String modell, int baujahr) {
      this.marke = marke;
      this.modell = modell;
      this.baujahr = baujahr;
}

   // Konstruktor mit Standardwert für baujahr
   public Auto(String marke, String modell) {
      this(marke, modell, 2025); // Ruft den anderen Konstruktor auf
   }

}

Durch Konstruktorüberladung wird die Initialisierung der Objekte flexibler und einfacher an verschiedene Anforderungen anpassbar.

5. Kapselung und das Prinzip der Datenkapselung

Ein wesentliches Prinzip der OOP ist die Kapselung (Encapsulation). Mit ihr werden die inneren Details eines Objekts verborgen und nur über definierte Schnittstellen (Methoden) zugänglich gemacht. Dies schützt die Daten vor ungewollten Zugriffen und Manipulationen.

In der Klasse Auto wird beispielsweise das Attribut marke als private deklariert:

private String marke;

Dadurch wird verhindert, dass außerhalb der Klasse direkt darauf zugegriffen wird. Stattdessen werden Getter- und Setter-Methoden angeboten, um kontrolliert Änderungen vorzunehmen:

public String getMarke() {
   return marke;
}

public void setMarke(String marke) {
   // Hier können zusätzliche Prüfungen eingebaut werden
   this.marke = marke;
}

Mit dieser Technik sorgt man dafür, dass die Objektdaten konsistent bleiben und nicht unbeabsichtigt verändert werden.

6. Modifikatoren, Getter und Setter

Modifikatoren wie private, public und protected steuern den Zugriff auf Klassenmitglieder:

  • public: Mitglieder sind von überall zugreifbar.
  • private: Mitglieder sind nur innerhalb der gleichen Klasse sichtbar.
  • protected: Mitglieder sind innerhalb der gleichen Klasse und deren Subklassen zugänglich.

Getter und Setter sind spezielle Methoden, die den kontrollierten Zugriff auf private Attribute ermöglichen. Sie sind nicht nur nützlich, um Datenlesefehler zu vermeiden, sondern auch um bei Bedarf Logik einzubauen (z. B. Validierungen):

public class Student {
   private int matrikelnummer;

   // Getter-Methode
   public int getMatrikelnummer() {
      return matrikelnummer;
   }

   // Setter-Methode
   public void setMatrikelnummer(int matrikelnummer) {
      // Beispiel: Prüfung, dass die Matrikelnummer positiv ist
      if(matrikelnummer > 0) {
         this.matrikelnummer = matrikelnummer;
      } else {
         System.out.println("Ungültige Matrikelnummer!");
      }
   }
}

Mit diesen Techniken können du und deine Mitstudierenden sicherstellen, dass die Datenstrukturen in euren Programmen robust und fehlerresistent bleiben.

Übungen

Aufgaben

  • Aufgabe 1: Implementierung einer Klasse
    • Erstelle eine Klasse „Auto“ mit den Attributen: marke, modell und baujahr.
    • Implementiere einen Konstruktor, der alle Attribute initialisiert.
    • Füge eine Methode hinzu, die eine einfache Nachricht ausgibt, z. B. „[Marke] [Modell] startet!“
    • Erstelle in einer Main-Methode ein Objekt der Klasse Auto und rufe die Methode auf.
  • Aufgabe 2: Implementierung von Kapselung
    • Nimm die Klasse „Student“ und deklariere die Attribute (z. B. name, matrikelnummer, studiengang) als private.
    • Implementiere jeweils passende Getter und Setter-Methoden.
    • Füge Logik in den Setter für die Matrikelnummer ein, sodass nur positive Werte akzeptiert werden.
    • Teste deine Implementierung in einer einfachen Main-Methode, indem du versuchst, ungültige Werte zu setzen, und überprüfe, ob deine Validierung wirkt.

Fragen

  1. Was versteht man unter einer Klasse in der objektorientierten Programmierung?
  2. Wie unterscheiden sich ein Konstruktor und eine normale Methode?
  3. Erkläre den Begriff der Kapselung und nenne zwei Vorteile, die sich daraus ergeben.
  4. Warum werden Attribute in der Regel als private deklariert und über Getter/Setter zugänglich gemacht?

Zusammenfassung

  • Eine Klasse ist ein Bauplan, der Attribute und Methoden definiert, während ein Objekt eine konkrete Instanz dieser Klasse darstellt.
  • Konstruktoren initialisieren Objekte und können überladen werden, um flexible Initialisierungen zu ermöglichen.
  • Kapselung verbirgt die internen Details eines Objekts und sorgt für kontrollierten Zugriff über Getter und Setter.
  • Zugriffsmodifikatoren (private, public, protected) steuern, wer welche Klassenmitglieder einsehen und verändern darf.
