Klassen und Objekte

Einführung – Von Bauplänen zu realen Objekten

Stell dir vor, du möchtest ein Auto bauen. Du hast einen exakten Bauplan, der alle Komponenten des Autos beschreibt – Motor, Karosserie, Räder und Innenausstattung. Dieser Bauplan enthält alle Informationen, die du benötigst, um ein Auto zu produzieren, jedoch ist er selbst noch kein fahrbares Fahrzeug. In der objektorientierten Programmierung (OOP) entspricht dieser Bauplan einer Klasse.

Einmal erstellt, kannst du anhand dieses Bauplans beliebig viele Autos (Objekte) fertigen – jedes Auto wird zwar nach demselben Konzept gebaut, besitzt jedoch seine eigenen Eigenschaften (z. B. Farbe, Baujahr) und verhält sich in bestimmten Situationen individuell. Genau so verhält es sich in der OOP: Eine Klasse definiert die Struktur und das Verhalten (Methoden), während Objekte konkrete Instanzen dieser Klasse sind. Diese Metapher – vom abstrakten Bauplan zum realen Produkt – soll dir helfen, den grundlegenden Zusammenhang zwischen Klassen und Objekten zu verstehen.

In diesem Kapitel wirst du lernen:

  • Was Klassen und Objekte sind
  • Wie du Klassen erstellst und Objekte instanziierst
  • Welche Rolle Attribute, Methoden und Konstruktoren spielen

Definition von Klassen

In Java beschreibt eine Klasse den Bauplan eines Datentyps. Eine Klasse enthält:

  • Attribute (Felder): Diese speichern den Zustand eines Objekts in Form von Daten. 
    • Bei einem Auto könnten das z. B. die Marke, das Modell und das Baujahr sein.
  • Methoden: Diese definieren das Verhalten, das Objekte ausführen können. 
    • Bei einem Auto könnte z. B. eine Methode zum Starten des Motors oder Bremsen gedacht sein.
  • Konstruktor(en): Der Konstruktor wird aufgerufen, wenn du basierend auf einer Klasse ein Objekt erstellen möchtest und definiert, was genau bei der Erstellung des Objekts passiert.
    • Bei einem Auto werden bei der Erstellung z.B. Marke, Modell und Baujahr festgelegt.

Die Definition einer Klasse legt also nicht nur fest, wie ein Objekt aussieht, sondern auch, welche Aktionen mit ihm ausgeführt werden können. Hier ein einfaches Beispiel einer Klasse „Auto“:

public class Auto {
   // Attribute – der Zustand eines Autos
   private String marke;
   private String modell;
   private int baujahr;

   // Konstruktor: Erzeugt ein neues Auto-Objekt
   public Auto(String marke, String modell, int baujahr) {
      this.marke = marke;
      this.modell = modell;
      this.baujahr = baujahr;
   }

   // Methode: Startet das Auto
   public void starten() {
      System.out.println("Das Auto startet.");
   }

   // Getter und Setter für das Attribut 'marke'
   public String getMarke() {
      return marke;
   }

   public void setMarke(String marke) {
      this.marke = marke;
   }

}

In diesem Beispiel fungiert die Klasse Auto als Bauplan. Sie legt fest, welche Eigenschaften (marke, modell, baujahr) ein Auto besitzt und welche Aktionen (z. B. starten) durchgeführt werden können.

Klassen und Dateinamen

Wenn du in Java eine öffentliche Klasse mit dem public Modifikator anlegst, muss die Datei denselben Namen haben wie die Klasse. Im obigen Beispiel haben wir die öffentliche Klasse Auto angelegt (public class Auto). Das bedeutet, dass die Datei, in der die Klasse gespeichert ist, Auto.java heißen muss.

Regel: Pro Datei darf es maximal eine public Klasse geben, die so heißt, wie die Datei. Die Datei kann aber weitere Klassen beinhalten, die nicht public sind.

// Wir befinden uns in der Datei Auto.java
public class Auto {
   /* Code für die Klasse Auto */
}

class Motorrad { // Das ist ERLAUBT
   /* Code für die Klasse Motorrad */
}

public class Fahrrad { // Das geht NICHT
   /* Code für die Klasse Fahrrad*/
}

Klasse oder Datei umbennenen

Es gibt immer wieder den Fall, dass man eine Klasse umbenennen möchte, z.B. weil man sich vertippt hat oder einem ein besserer Name eingefallen ist. Angenommen, ich möchte in unserem Fall meine Klasse doch lieber PKW anstelle von Auto nennen. In diesem Fall kann ich umständlich erst die Klasse und dann die Datei umbenennen. Nicht wundern: Wenn du das so machst erhältst du temporär eine Fehlermeldung, dass Klassenname und Dateiname nicht übereinstimmen.

Viel sinnvoller ist es allerdings IntelliJ die Arbeit machen zu lassen! Dafür klicke ich entweder in meinem Code auf den Namen meiner Klasse oder in der linken Spalte auf den Namen meiner Datei mit der rechten Maustaste. Dann öffnet sich ein Kontextmenü und dort klicken wir auf Rename. In diesem Fall benennt IntelliJ deine Klasse an allen Stellen in deinem gesamten Projekt um. Du musst also nicht selber umständlich suchen, wo du überall den Namen deiner Klasse verwendet hast und brauchst keine Angst haben, etwas zu übersehen. Das ist gerade dann sinnvoll, wenn du in einem großen Softwareprojekt etwas umbenennen möchtest. 

Erstellen von Objekten

Sobald eine Klasse definiert ist, kannst du Objekte davon erstellen.  Ein Objekt ist eine konkrete Ausprägung der Klasse, vergleichbar mit einem fertigen Auto, das auf Basis des Auto-Bauplans hergestellt wurde. Objekte werden auch als Instanzen bezeichnet. In der Informatik beschreibt eine Instanz eine konkrete Umsetzung oder ein Exemplar eines Modells. Da Objekte auch als Instanzen bezeichnet werden, wird der Vorgang des Erstellens eines Objekts auch als Instanziierung bezeichnet.

So könnte in der Main-Methode eines Programms ein Auto-Objekt erzeugt und verwendet werden:

public class Main {
   public static void main(String[] args) {
      // Erzeugen eines Auto-Objekts durch Aufrufen des Konstruktors
      Auto meinPrivatAuto = new Auto("VW", "Golf", 2020);
      meinPrivatAuto.starten(); // Ruft die Methode 'starten' auf

      // Erzeugen eines weiteren Auto-Objekts durch Aufrufen des Konstruktors
      Auto meinArbeitsAuto = new Auto("VW", "C-Klasse", 2021);

      // Marke des Arbeitsautos anpassen
      meinArbeitsAuto.setMarke("Mercedes");

      System.out.println("Marke: " + meinPrivatAuto.getMarke()); // Ausgabe: VW
      System.out.println("Marke: " + meinArbeitsAuto.getMarke()); // Ausgabe: Mercedes
   }
}

Durch den Konstruktor der Klasse werden die Objekte meinPrivatAuto und meinArbeitsAuto erzeugt und mit individuellen Eigenschaften ausgestattet. Eine Veränderung an meinArbeitsAuto hat keinerlei Auswirkung auf meinPrivatAuto, da es sich hier um 2 vollkommen autarke (unabhängige) Objekte handelt.

Konstruktoren

Ein Konstruktor ist eine spezielle Methode, die beim Erzeugen eines Objekts aufgerufen wird, um es zu initialisieren. Da es sich bei einem Konstruktor um eine Methode handelt, ist er wie eine Methode aufgebaut. Bei der Erstellung eines Konstruktors musst du folgende Dinge beachten:

  • Namen: Der Konstruktor hat immer denselben Namen wie die Klasse.
  • Parameter: Daten, welche zur Erstellung eines Objekts benötigt werden, können als Parameter an den Konstruktor übergeben werden
  • Keine Rückgabe: Konstruktoren haben keinen Rückgabetyp, auch nicht void. Das bedeutet, vor dem Namen steht kein Datentyp und nicht void. 

Das Schlüsselwort „this“

Das Schlüsselwort this bezieht sich auf das aktuelle Objekt in einer Methode oder einem Konstruktor.

Die häufigste Verwendung des Schlüsselworts this besteht darin, die Verwechslung zwischen Klassenattributen und Parametern mit demselben Namen zu beseitigen (da ein Klassenattribut durch eine Methode oder einen Konstruktorparameter überschattet wird). Wenn man das Schlüsselwort this in folgendem Beispiel in Zeile 7 und 8 weglässt, wäre die Ausgabe „Meine Position ist x:0 y:0“ statt „Meine Position ist x:34 y:65“.

class Koordinaten {

   public int x;
   public int y;
   
   public Koordinaten(int x, int y) {
      this.x = x;
      this.y = y;
   }
}

public class Main {

  public static void main(String[] args) {
    Koordinaten meinePosition = new Koordinaten(34, 65);
    System.out.println("Meine Position ist x:" + meinePosition.x + " y:" + meinePosition.y);
  }
}

this kann auch verwendet werden, um:

  • den Konstruktor der aktuellen Klasse aufzurufen
  • eine Methode der aktuellen Klasse aufzurufen
  • das aktuelle Objekt der Klasse zu erhalten

Überladung von Konstruktoren

Es ist möglich, mehrere Konstruktoren mit unterschiedlichen Parameterlisten zu definieren, um Objekte flexibel initialisieren zu können.

public class Auto {
   private String marke;
   private String modell;
   private int baujahr;

   // Konstruktor mit allen Parametern
   public Auto(String marke, String modell, int baujahr) {
      this.marke = marke;
      this.modell = modell;
      this.baujahr = baujahr;
}

   // Konstruktor mit Standardwert für baujahr
   public Auto(String marke, String modell) {
      this(marke, modell, 2025); // Ruft den anderen Konstruktor auf
   }

}

Durch Konstruktorüberladung wird die Initialisierung der Objekte flexibler und einfacher an verschiedene Anforderungen anpassbar.

Attribute und Methoden

Attribute und Methoden bilden das Herzstück einer Klasse, sie werden auch als die Eigenschaften einer Klasse bzw. eines Objekts bezeichnet.

  • Attribute (Felder) definieren den Zustand. Sie können verschiedene Datentypen haben (z. B. int, String, boolean).
  • Methoden definieren das Verhalten eines Objekts. Sie enthalten Code, der Operationen auf den Attributen oder sonstige Aktionen durchführt.

Ein anschauliches Beispiel:
Stell dir einen „Student“ vor. Er hat Attribute wie Name, Matrikelnummer und Studiengang. Gleichzeitig kann er Aktionen ausführen wie zur Vorlesung gehen oder sich für Prüfungen anmelden. In Java könnte dies folgendermaßen modelliert werden:

public class Student {
   private String name;
   private int matrikelnummer;
   private String studiengang;

   // Konstruktor
   public Student(String name, int matrikelnummer, String studiengang) {
      this.name = name;
      this.matrikelnummer = matrikelnummer;
      this.studiengang = studiengang;
   }

   // Eine Methode: Der Student meldet sich an
   public void anmelden() {
      System.out.println(name + " hat sich für den Kurs angemeldet.");
   }

   // Getter und Setter
   public String getName() {
      return name;
   }

   public void setName(String name) {
      this.name = name;
   }

}

Diese Trennung von Zustand (Attribute) und Verhalten (Methoden) ist ein zentrales Merkmal der Objektorientierten Programmierung. Durch die klare Strukturierung kann der Code übersichtlicher und modularer gestaltet werden.

Statische Eigenschaften einer Klasse

Die Eigenschaften einer Klasse können mit dem Modifikator static versehen werden. Auf ein Attribut oder eine Methode, die static ist, kann zugegriffen werden, ohne dass ein Objekt von der Klasse erstellt werden muss. Man greift dann über die Klasse und nicht über das Objekt auf die Eigenschaft zu. Statische Eigenschaften werden verwendet, wenn diese Eigenschaften eher zur Klasse, als zum Objekt gehören bzw. nicht einem konkreten Objekt zugeordnet werden. Dazu gehören zum Beispiel folgende Methoden und Eigenschaften aus der Math-Klasse:

Math.sin(3.64); // Berechnung des Sinus 
Math.max(3, 7); // Ermittlung des Maximalwerts
System.out.println(Math.PI); // Ausgabe von PI

Auf die Methoden und Attribute der Math-Klasse kannst du zugreifen, ohne ein Objekt erstellen zu müssen. Sie werden in der Math-Klasse einfach nur gruppiert. Du selber kannst in deinen eignen Programmen Methoden, die du schreibst und die zu keinem Objekt gehören, in einer eigenen Klasse gruppieren, um einfach auf diese zugreifen zu können.

Ein weiterer Vorteil von statischen Eigenschaften ist, dass du auf sie zugreifen kannst, bevor du ein Objekt erstellst und, dass sich Änderungen statischen Attributen, auf alle Objekte auswirken. Stell dir vor, wir haben eine Klasse für eine Semestergruppe. Die maximale Teilnehmerzahl für eine Semestergruppe liegt bei 15 Studierenden. Diese Eigenschaft gehört eher zu der Klasse Semestergruppe, anstatt zu jeder einzelnen Semestergruppe individuell. Daher kann man dieses Attribut statisch machen. Falls sich die Zahl doch einmal ändern sollte, kann man sie einfach in der Klasse ändern und diese Auswirkung betrifft dann alle Objekte.

class Semestergruppe {
    
    // Öffentliches statisches Attribut
    public static int MAX_TN = 15;
    private int tn;
    private String id;
    
    Semestergruppe (int tn, String id) {
        if(tn > MAX_TN) {
            System.out.println("Fehler bei der Teilnehmerzahl! Setze Anzahl auf " + MAX_TN);
        } else {
            this.tn = tn;
        }
        this.id = id;
    }
}

public class Main {
    public static void main(String[] args) {
        // Zugriff auf das Attribut MAX_TN über den Klassennamen
        System.out.println("Maximale Anzahl an Studierenden pro Gruppe: "+Semestergruppe.MAX_TN);
    }
}

Zusammenfassung

  • Eine Klasse ist ein Bauplan, der Attribute und Methoden definiert, während ein Objekt eine konkrete Instanz dieser Klasse darstellt.
    • Die Attribute und Methoden werden als die Eigenschaften einer Klasse bezeichnet
  • Die Eigenschaften einer Klasse können statisch oder nicht statisch sein
    • Dies wird mithilfe des Modifikators static festgelegt
    • Statische Eigenschaften gehören zur Klasse und es kann auf sie zugegriffen werden, ohne ein Objekt der Klasse erstellen zu müssen.
    • Nicht statische Eigenschaften können nur über ein Objekt der Klasse verwendet werden.
  • Konstruktoren initialisieren Objekte und können überladen werden, um flexible Initialisierungen zu ermöglichen.
