Grundlagen

    Einführung in Java

    In diesem Abschnitt möchten wir uns ansehen, wie man in Java Code schreibt. Vieles davon wird identisch oder ähnlich zu dem sein, wie es in Processing ist. Dennoch schauen wir uns alles einmal an, damit wir wissen, wie man in Java programmiert. Die einzelnen Themen werden nur kurz angeschnitten und nicht ausführlich erklärt, da diese Grundlagen vorausgesetzt werden.

    In diesem Abschnitt lernst du:

    • Die Grundlagen von Java
      • Syntax, Datentypen, Konsolenausgabe, Kontrollstrukturen, Arrays, Methoden und Math.
    • Die Unterschiede und Gemeinsamkeiten von Processing und Java

    Syntax

    Die Syntax von Java ist identisch mit dem von Processing. Hier einige wichtige Punkte:

    • Groß- und Kleinschreibung muss beachtet werden
    • Semikolons (;) beenden Anweisungen.
    • Blöcke werden durch geschweifte Klammern {} definiert.
    • Bezeichner
      • dürfen nicht mit einem Schlüsselwort der Sprache Java identisch sein, z.B. darf man nicht int oder float als Variablenname verwenden
        bestehen aus Ziffern (0-9), Buchstaben (a-z, A-Z) und zwei weiteren Zeichen (Unterstrich und $)
        dürfen nicht mit einer Ziffer beginnen
        dürfen keine Leerzeichen enthalten und auch keinen Bindestrich (würde als Minus interpretiert werden)
    • Kommentare können einzeilig oder mehrzeilig sein.
      • Einzeilig: // Kommentar
      • Mehrzeilig:
        /*
           Mehrzeiliger Kommentar
        */

    Datentypen

    Primitive Datentypen

    Java unterscheidet zwischen primitiven Datentypen und Referenzdatentypen. Primitive Datentypen zeichnen sich dadurch aus, dass sie einfach nur einen Wert speichern (also eine Zahl oder einen Wahrheitwert). Sie werden daher auch als Wertetypen bezeichnet. Wertetypen kannst du daran erkennen, dass der Datentyp klein geschrieben wird. Dies sind die 8 primitiven Datentypen in Java:

    Primitiver Typ Beschreibung Beispiel
    byte, short, int, long Ganze Zahlen (unterschiedlicher Wertebereich) int x = 10;
    float, double Gleitkommazahlen (unterschiedlicher Wertebereich) double y = 5.5;
    char Einzelne Zeichen char c = 'A';
    boolean Wahrheitswerte boolean b = true;

    Alle anderen Datentypen sind Referenztypen. Diesen Namen haben sie, weil sie keine einfachen Werte speichern, sondern eine Referenz (so etwas wie ein Link) zu einer Stelle im Speicher, wo dann ein Objekt gespeichert ist, dass aus mehreren Daten und Methoden besteht. Aus diesem Grund werden Referenztypen auch als Objekttypen bezeichnet. Objekttypen erkennst du daran, dass sie groß geschrieben werden. Objekte werden mithilfe von Klassen ("Bauplänen") erstellt. Das haben wir bereits in der Einführung gelernt. Was man sonst noch alles über Refenztypen bzw. Objekttypen wissen muss, lernen wir also sobald wir uns dem Thema "Klassen und Objekte" widmen.

    Enum

    Ein enum (kurz für "enumeration" = Aufzählung) ist eine besondere Klasse und repräsentiert eine Gruppe von Konstanten, die wir selber festlegen können. Um einen enum zu erstellen, nutzt du das Schlüsselwort enum, wählst einen Bezeichner und ergänzt einen Codeblock. Innerhalb des Codeblocks kannst du deine Konstanten auflisten und diese mithilfe von Kommata trennen. Auf diese Konstanten kannst du dann zugreifen mithilfe der Punktnotation:

    enum Day {
    SUNDAY, MONDAY, TUESDAY, WEDNESDAY,
    THURSDAY, FRIDAY, SATURDAY 
}

System.out.println(Day.MONDAY);

    Du solltest enum-Typen immer dann verwenden, wenn du einen festen Satz von Konstanten darstellen willst. Dazu gehören natürliche enum-Typen wie die Planeten in unserem Sonnensystem und Datensätze, bei denen du zu Programmstart alle möglichen Werte kennst (z.B. Auswahlmöglichkeiten in einem Menü).

    Hinweis: Gemäß algemeiner Programmierkonventionen werden Konstanten immer in GROßBUCHSTABEN geschrieben und Leerzeichen mit einem Unterstrich _ markiert.

    Typumwandlung

    Eine Typumwandlung wird durchgeführt, wenn du den Wert eines primitiven Datentyps einem anderen primitiven Datentyp zuweist. In Java gibt es 2 Arten der Typumwandlung:

    • Erweiternde Typumwandlung (automatisch)
      • In diesem Fall weißt du einen kleineren Datentyp einem größeren Datentyp zu
      • byte -> short -> char -> int -> long -> float -> double
    • Reduzierende Typumwandlung (manuell)
      • In diesem Fal weißt du einen größeren Datentyp einem kleineren zu.
      • Dies erfolgt manuell, indem der Zieldatentyp in Klammern () vor dem Wert hinzugefügt wird.
      • double -> float -> long -> int -> char -> short -> byte

    Beispiele

    byte myByte = 3;
float myFloat = myByte; // Automatische Typumwandlung, ergibt 3.0

double myDouble = 14.845;
int myInt = (int) myDouble; // Manuelle Typumwandlung, ergibt 14

    Konsolenausgabe

    Eine einfache Konsolenausgabe erfolgt in Java mit System.out.println().  System ist eine Klasse innerhalb von Java, welche viele wichtige Methoden und Daten bereithält. Sie enthält unter anderem den Abschnitt out (kurz für Output), welcheunter anderen folgende Methoden enthält:

    • println(): Kurz für print line - erstellt eine Konsolenausgabe mit Zeilenumbruch am Ende
    • print(): Erstellt eine Konsolenausgabe ohne Zeilenumbruch am Ende
    • printf(): Erstellt eine formatierte Konsolenausgabe (vgl. String.format())

    Kontrollstrukturen

    Kontrollstrukturen steuern den Ablauf deines Programms. Die wichtigsten Kontrollstrukturen sind Fallunterscheidungen (if und switch) und Schleifen (for, while, do-while).

    if-Anweisung

    Mithilfe einer if-Anweisung kann eine Fallunterscheidung druchgeführt werden. In den gescheiften Klammern mit eine Bedinung angegeben werden (z.B. in Form eines Vergleichs oder einer booleschen Variable). Eine differenzierte Fallunterscheidung erfolgt mithilfe von else und else if.

    int x = 10;
if (x > 5) {
   System.out.println("x ist größer als 5");
} else {
   System.out.println("x ist kleiner oder gleich 5");
}

    Switch

    Mithilfe eines Switchs kann eine Fallunterscheidung durchgeführt werden, wenn eine Variable auf einen bestimmten Wert überprüft werden soll und die möglichen Optionen bekannt sind.
    int tag = 3;
        
switch (tag) {
   case 1:
      System.out.println("Montag");
      break;
   case 2:
      System.out.println("Dienstag");
      break;
   case 3:
      System.out.println("Mittwoch");
      break;
   case 4:
      System.out.println("Donnerstag");
      break;
   case 5:
      System.out.println("Freitag");
      break;
   case 6:
      System.out.println("Samstag");
      break;
   case 7:
      System.out.println("Sonntag");
      break;
   default:
      System.out.println("Ungültiger Tag");
   }
}



    For-Schleife

    For-Schliefen sind Zählschleifen, die im Schleifenkopf eine Laufvariable, ein Abbruchkriterium und eine Schleifenaktion festlegen. Eine Schleife kann mithilfe von break abgebrochen werden. Ein Schleifendurchlauf kann mithilfe von continue übersprungen werden.

    for (int i = 0; i < 10; i++) {
   System.out.println(i);
}

    While-Schleife

    While-Schleifen sind kopfgesteuerte Schleifen, welche ein Abbruchkriterium im Schleifenkopf enthalten.

    int i = 0;
while (i < 10) {
   System.out.println(i);
   i++;
}

    Do-while-Schleife

    Do-While-Schleifen sind fußgesteuerte Schleifen, welche ein Abbruchkriterium im Schleifenfuß enthalten.

    int i = 0;
do {
   System.out.println(i);
   i++;
} while (i < 10)

    try-Catch

    Ein try-Catch-Block ist eine Struktur, um Ausnahmen (Exceptions) abzufangen:

    int a = 3;
int b =  0;

try {

   System.out.println(a / b);

} catch (ArithmeticException e) {

   System.out.println("Division durch 0 nicht möglich!");

}
    Das Thema Ausnahmebehandlung wird im Abschnitt "Ausnahmebehandlung" ausführlicher erklärt.

    Arrays

    Arrays sind Sammlungen von Elementen desselben Datentyps. Arrays können mehrere Dimensionen haben. Das bedeutet, dass in einem Array-Element ein weiterer Array gespeichert ist. Arrays haben in der ersten Dimension eine feste Länge, welche bei der Deklaration angegeben werden muss. Mithilfe des Index, kann auf einzelne Elemente zugegriffen werden. Der Index beginnt bei 0. Bei Arrays handelt es sich um Objekttypen. 

    // Kurze Schreibweise
int[] zahlen = {1, 2, 3, 4, 5};
System.out.println(zahlen[0]); // Gibt "1" aus

// Lange Schreibweise
int[][] werte = new int[2][2];
werte[0][0] = 1;
werte[0][1] = 2;
werte[1][0] = 3;
werte[1][1] = 4;

System.out.println(werte[0][1]); // Gibt "2" aus

    Methoden

    Methoden sind sind Codeblöcke, welche nur dann ausgeführt werden, wenn sie aufgerufen werden. Methoden werden auch häufig als Funktionen bezeichnet. Methoden werden verwendet, um Code zu strukturieren und Code wiederzuverwenden. Sobald eine Reihe von Operationen mehrmals ausgeführt werden soll, ist es sinnvoll sie in einer Methode zusammenzufassen. Methoden können Daten entgegennehmen in Form von Parametern. Außerdem können sie Daten zurückgeben in Form von Rückgabewerten.

    Wichtig: In Java können Methoden nur innerhalb von Klassen verwendet werden!

    public class Main {
  public static void main(String[] args) {

    // Aufruf der Methode ohne Parameter und Rückgabewert
    gruesseWelt();

    // Aufruf der Methode addiere mit zwei Parametern
    int ergebnis = addiere(5, 3);
    System.out.println("Das Ergebnis der Addition ist: " + ergebnis);
  }

  // Definition einer einfachen Methode ohne Parameter und Rückgabewert
  public static void gruesseWelt() {
    // Diese Methode gibt einfach einen Text aus
    System.out.println("Hallo Welt!");
  }

  // Definition einer Methode mit zwei Parametern und einem Rückgabewert
  public static int addiere(int zahl1, int zahl2) {
    // Diese Methode addiert zwei Zahlen und gibt das Ergebnis zurück
    int summe = zahl1 + zahl2;
    return summe; // Rückgabe des Ergebnisses
  }
}

    Überladung

    Methoden können in Java überladen werden. Das bedeutet, dass mehr als eine Definition für diese Methode vorliegt. Diese Definitionen unterscheiden sich anhand der Parameter (Anzahl und Reihenfolge). Methoden-Überladung wird häufig eingesetzt, um Standardwerte zu setzen, wenn z.B. keine Daten vorliegen.

    public class Main {
  public static void main(String[] args) {

    // Aufruf der Methode ohne Parameter 
    System.out.println(hallo()); // Ausgabe: Hallo du!
    // Aufruf der Methode ohne Parameter 
    System.out.println(hallo("Ingrid")); // Ausgabe: Hallo Ingrid!
  }

  // Definition einer Methode ohne Parameter
  public static String hallo() {
    return "Hallo du!"; // Rückgabe der Begrüßung mit Name
  }

  // Definition einer Methode mit Parameter
  public static String hallo(String name) {
    return "Hallo " + name + "!"; // Rückgabe der Begrüßung mit Name
}

    Modifikatoren

    Modifikatoren werden in Java verwendet, um die Sichtbarkeit, Lebensdauer und Veränderbarkeit von Elementen festzulegen. Ein paar Modifikatoren sind und bereits begegnet, wie z.B. final, public und static. Modifikatore lassen sich in 2 Gruppen einteilen: Zugriffsmodifikatoren und Nicht-Zugriffsmodifiaktoren.

    Zugriffsmodifikatoren

    Zugriffsmodifikatoren steuern die Sichbarkeit und Klassen, Methoden und Attributen. In Java gibt es 4 Zugriffsmodifkatoren:

    Modifikator Beschreibung
    public (öffentlich) Diese Klasse (bzw. das Attribut, die Methode) ist von jeder anderen Klasse aus zugänglich.
    protected (geschützt) Diese Klasse (bzw. das Attribut, die Methode) ist innerhalb seines eigenen Pakets und von Unterklassen zugänglich.
    private (privat) Diese Klasse (bzw. das Attribut, die Methode) ist nur innerhalb seiner eigenen Klasse zugänglich.
    default (kein Schlüsselwort) Diese Klasse (bzw. das Attribut, die Methode) ist nur innerhalb seines eigenen Pakets zugänglich. 
    // Öffentliche Zahl
public class Beispiel {
    public int oeffentlicheZahl;
    protected int geschuetzteZahl;
    int defaultZahl; // Standard Zugriff
    private int privateZahl;
}

    In diesem Beispiel ist oeffentlicheZahl von überall zugänglich, geschuetzteZahl ist innerhalb des Pakets und der Subklassen zugänglich, defaultZahl ist innerhalb des Pakets zugänglich und privateZahl ist nur innerhalb der Klasse Beispiel zugänglich.

    Nicht-Zugriffsmodifikatoren

    Nicht-Zugriffsmodifikatoren bieten zusätzliche Funktionen wie die Kontrolle der Vererbung, die Gewährleistung der Unveränderlichkeit und die Definition besonderer Eigenschaften.

    Für Klassen gibt es die Modifikatoren final und abstract:

    Modifikator Beschreibung
    final (final)

    Die Klasse kann nicht von anderen Klassen vererbt werden.

    Mehr zumThema Vererbung lernen wir im Kapitel Vererbung.
    abstract (abstrakt)

    Die Klasse kann nicht verwendet werden, um Objekte zu erstellen (Um auf eine abstrakte Klasse zuzugreifen, muss sie von einer anderen Klasse vererbt werden.

    Mehr zum Thema Vererbung und Abstraktion lernen wir im Kapitel Vererbung und Abstrakte Klassen und Interfaces)

    Für Attribute und Methoden gibt es folgden Modifikatoren:

    Modifikator Beschreibung
    final (final) Attribute und Methoden können nicht überschrieben/verändert werden
    static (statisch) Attribute und Methoden gehören zur Klasse, nicht zum Objekt
    abstract (abstrakt)

    Kann nur in einer abstrakten Klasse verwendet werden und kann nur auf Methoden verwendet werden. Die Methode hat keinen Codeblock, zum Beispiel abstract void run();. Der Codeblockwird in der Subklasse erstellt.

    Mehr zum Thema Vererbung und Abstraktion lernen wir im Kapitel Vererbung und Abstrakte Klassen und Interfaces)
    transient (transient) Verhindert die Serialisierung von Klassenfeldern.
    synchronized (synchronisiert) Wird im Multithreading verwendet, um sicherzustellen, dass immer nur ein Thread auf eine Methode oder einen Block zugreifen kann.
    volatile (flüchtig) Stellt sicher, dass der Wert einer Variablen immer aus dem Hauptspeicher gelesen wird.

    Hinweis: Die Themen Serialisierung und Multithreading werden wir in diesem Modul nicht behandeln.

    public class Beispiel {
    public static final int KONSTANTE = 100;

    public static void main(String[] args) {
        System.out.println("Constant Value: " + CONSTANT);
    }
}

    Math

    Die Klasse Math enthält viele Methoden mit deren Hilfe mathematische Operationen durchgeführt werden können. Hier sind einige Beispiele:

    Methode Beschreibung Rückgabetyp
    abs(x) Gibt den absoluten Wert von x zurück double|float|int|long
    atan2(y,x) Gibt den Winkel Theta aus der Umwandlung von rechtwinkligen Koordinaten (x, y) in Polarkoordinaten (r, Theta) zurück. double
    ceil(x) Gibt den nach oben als Ganzzahl gerundeten Wert von x zurück double
    cos(x) Gibt den Cosinus von x (im Bogenmaß) zurück double
    floor(x) Gibt den nach unten als Ganzzahl gerundeten Wert von x zurück double
    max(x, y)  Gibt den größeren Wert  zurück double|float|int|long
    min(x, y) Gibt den kleineren Wert zurück double|float|int|long
    pow(x, y) Gibt denWert von x hoch y zurück double
    random() Gibt eine Zufallszahl zwischen 0 und 1 zurück double
    round(x) Gibt den als Ganzzahl gerundeten Wert von x zurück long|int
    sin(x) Gibt den Sinus von x (x is im Bogenmaß) zurück double
    sqrt(x) Gibt Wurzel von x zurück double
    tan(x) Gibt den Tangens x (in Bogenmaß) zurück double

    Zusammenfassung

    • Die Syntaxregeln in Java sind identisch mit den Regeln von Processing und müssen zwingend eingehalten werden, damit dein Programm ausführbar ist.
    • In Java werden Datentypen unterschieden nach primitiven Datentypen und Referenztypen.
      • Primitve Datentypen speichern einfache Werte und werden klein geschrieben. Daher werden sie auch als Wertetyp bezeichnet
      • Referenztypen speichern Objekte und werde groß geschrieben. Daher werden sie auch als Objekttyp bezeichnet.
    • Mithilfe des Datentyp enum kann ein Set von Konstanten definiert werden.
    • Die Konsolenausgabe erfolgt in Java mithilfe von System.out.println() (alternativ geht auch print() oder printf() am Ende)
    • In Java gibt es 2 Arten der Typumwandlung für pirmitive Datentypen
      • Die erweiternde Typumwandlung in einen größeren Datentyp erfolgt autmatisch.
      • Die reduzierende Typumwandlung in einen kleineren Datentyp erfolgt manuell, in dem der Ziel-Datentyp in Klammern () vor den zu konvertierenden Wert gesetzt wird.
    • In Java gibt es Fallunterscheidungen und Schleifen
      • Fallunterscheidung: if und switch
      • Schleifen: for, while und do-while
      • Diese Kontrollstrukturen sind identisch mit denen in Processing.
      • Ausnahmebehandlungen funktionieren in Java, so wie in Processing, allerdings steigen wir in diesem Skript etwas tiefer in das Thema ein.
    • Arrays sind Sammlungen gleichartiger Elemente und müssen in ihrem Typ definiert werden.
      • Es gibt ein-und mehrdimensionale Arrays
      • Auf die einzelnen Elemente kann mithilfe des Index zugegriffen werden, welcher bei 0 beginnt
      • Arrays in JAva sind identisch mit Arrays in Processing
    • Methoden sind Gruppen von Befehlen, die in einem Coeblock zusammengefasst werden, um Struktur und Wiederverwendbarkeit zu schaffen.
      • Methoden haben wir in Processing immer als Funktionen bezeichnet, sind aber dasselbe
      • In Java können Methoden nur innerhalb von Klasser erstellt werden.
    • Die Klasse Math bietet eine Reihe von vorgefertigten Befehlen, mit denen gängige arithmetische Operationen durchgeführt werden können.
